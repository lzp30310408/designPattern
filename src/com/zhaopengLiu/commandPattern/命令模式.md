**命令模式**
       
    将一个请求封装为一个对象，使发出请求的责任和执行请求的责任分割开，两者之间通过命令进行沟通
    方便对于命令的存储，传递，调用，增加
    
**优点**
   
    降低系统的耦合度，将操作的对象和实现该操作的对象解耦
    增加或删除命令很方便，符合“开闭原则”
    可以实现宏命令，将命令模式和组合模式想结合，多个命令组合成一个命令，即宏命令
    可以实现Undo和Redo的操作，可以实现命令的操作和恢复
   
**缺点**
    
    产生大量具体命令类，增加系统的设计复杂度。
    
**结构**
    
    接收者A<---具体命令A  具体命令B --> 接受者B
                ^        ^
                |        |
                |        |
    调用者 --->    抽象命令
