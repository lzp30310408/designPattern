**代理模式**
       
    在某些时候，一个对象需要控制访问的权限，所以只能提供该对象的代理供调用方来处理
    
**优点**
    
    保护对象；拓展功能；客户与目标对象分离，降低耦合度
   
**缺点**

    添加代理对象，造成速度减慢
    造成系统复杂度提升
    
**结构**
    
    客户 --->    抽象产品
                ^     ^
                |     |
                |     |
            代理产品   真实产品
            
例：spring动态代理
